2.1 Settings

The following settings can be carried out in the administration menu of the forum under "settings":


Forum Name
Here you can give your forum a name.

Language
Here you set the default-language of your forum.

Theme
Here you set the default theme of your forum. If you choose "default", forum will use colors of the pragmaMx theme.
Please note: The default-theme is handled by the "defaulttheme.php" and have nothing to do with the "default"-forumtheme you can find in the admin menu under themes. This "default"-theme is only a "space holder" so that you can choose a "default"-theme.

Should the subject of the last modified thread been shown as forums-lastpost?
If active, a short version of the subject from the last modified or generated thread will be shown as forums-lastpost. The next setting also belong to this.

How many chars of the subject should be shown?
How many chars of the subject should be shown in the forums-view?

Activate the possibility to link forums in other forums
If you activate this option, you can find in the administrator menu under "forums" a point that allows you to link a forum into another one. Linked forums are displayed with an animated link-picture. By using this function you increase total process time of your forum. How much this is depends on your system. Best is to make a test once with this option enbabled, second time with the option disabled.
It is normal that linked forums are displayed after the "normal" subforums at the end of the page. If you set e.g. 25 forums to be displayed on one page, then there will be ALSO 25 linked forums been displayed on this page (if so many forums are linked into this forum), so that the total number then is 50!

Activate the possibility to link threads in other forums
Whis this option team-members have the possibility to link threads to another forum so that they are accessible from more than one forum. With activating this option total prozessing time of your forum will increase. How much this is depends on your system. Please have a look onto the total prozessing time at the bottom of the pages to find it out for you.
It is normal that linked threads are displayed after the "normal" threads at the end of the page. If you set e.g. 25 threads to be displayed on one page, then there will be ALSO 25 linked threads been displayed on this page (if so many threads are linked into this forum), so that the total number then is 50!

Posts Per Page
How many Posts (answers) will per default be displayed on one page. Every user can change this value for itself in its profile settings.

Topics Per Page
How many threads (contributions, Threads) will per default be displayed on one page. Every user can change this value for itself in its profile settings.

Members Per Page
How many members should be displayed on one "memberlist"-page

Replies required for hot topic
Here you adjust how many answers to a contribution are required so that this contribution gets the status "Hot Topic".

Seconds to wait until next post is allowed (flood control)
Here is set the time in seconds before a user can transmit its next contribution. This makes it more difficult to "spam" the forum, e.g. with the reload-function of the browser.

Board Status
You hereby can deactivate the forum. The forum is then if e.g. maintenance work is executed only accessible for administrators. Normal users are shown the text which is written in the text field below (if the forum is deactivated).

Who's online in index
If this function is activated, on the mainpage of the forum (and the center block) is show which members are online in the forum.

should advanced online-information been showed in the header
This option is only available if "Who's online in index" is active

This advanced option shows a kind of "tracking list" accessible over a link in the header of the forum. On this list you can see who is in which forum online and how long.

Allow only registered users to view boards
If this option is activated, only regged users of the pragmaMx system can access the forum.

Categories Only On Main Page
If you activate this, on the index-page of the forum no forums will be shown, only a list of you different categories.

Private forums hidden to unauthorized users
If this is active, user can only see forums to which they have access.

Show sorting options on forumdisplay
User will see sorting-options at the bottom of the forumdisplay-page if active.

Board Rules
When you activate this, you will get a button at the bottom of you pages "board rules". When you click on it a popup-window will open showing the board rules from the textfield below.

Search Status
If option is on, users are allowed to seach into the forums threads and posts.

Display Today Messages
If active, a link in the header will show all messages of the day. You can adjust the day-range below

Number of days for "Messages of the day" (in days)
Here you set the range of days which is shown in the "Messages of the day"-page

FAQ Status
If active, FAQ-link is shown in the header. Here users can get more information about BB-rules and member-status.

Member List Status
If active, a link is shown in the header of the forum (and the center-block) which displays a forum member list

show anonymous users the member list
Only available, if "Member List Status" is active

If this option is turned off, no anonymous user can access the member list

Stats Status
If active, users can access advanced statistics of your forum.

Rows number in stats
How many lines of each categorie should be shown in the statistics.

Stats on Index: (## topics/ ## posts/ ## members)
Shows short-stats on the mainpage of the forum.

Display Navigation Jumper
Shows a Quick-Jumper at the bottom of each page. With this, you can easily jump between the forums. It is actuallized every time you change something in the "forums"-option of the administrator menu.

use an dynamic navigation quick-jumper (saver, but a little bit slower)
Only available, if "Display Navigation Jumper" is active.

What does dynamic means? This jumper only contains the forums as jump-target, the user is allow to access. The "static" jumper contains all forums of the forum, also staff and private once.

Post Icon Status
If active, users can choose an icon for their threads and posts.

How many lines and rows of smilies should been displayed when writing posts
Here you set how many lines and rows of Smilies are displayed when writing a thread or a post. All other Smilies will be accessible over the "more smilies ...."-option.

Avatar Status
If this is active, user avatars will be show in the "threadview"

Allow normal members to set topic-priority
If active, all users can set the priority of a thread. Otherwise only team-members can set the priority of a thread.

HINT: if there are now different colors shown when changing the priority of a thread, the "style.css" of you pragmaMx system will filter it out. So you have to modify the "style.css" of the system, use another theme or give the forum it's own "style.css"-file.

Allow members to use the Private Messages Service
With this option member have the possibility to send private messages to the author of a thread or post by using the "u2u"-button.

Report Post Status
If this is active, users can report new posts to the author of a thread and the moderator of the forum.

Processing Time
Shows total processing time at the bottom of the pages.

Admin stars Management
Here you can choose if an administrator is handled as a normal user with the stars or if administrators should allways have three stars more than the highest user-rank.

Administrator E-Mail
Here you set the "from"-address for emails delivered by the forum

Notification status for administrators
This option activates the notify-service for administrators.

Send a mail on every ... ( four options)
Here you can enable or disable notify-service for different kind of changes in the forum. Each administrator has to set in his profile too: if he wants to be notified and in which cases he or she would like to be notified. These settings here are global that means, if they are switched off, administrators can't use the disabled notify-cases. This might help you to reduce your traffic.

Notification status for moderators
This option activates the notify-service for moderators.

Permit moderators notification on each ... ( four options)
Here you can enable or disable notify-service for different kind of changes in the forum. Each moderator has to set in his profile too: if he wants to be notified and in which cases he or she would like to be notified. These settings here are global that means, if they are switched off, moderators can't use the disabled notify-cases. This might help you to reduce your traffic.

Size of post message box
With this settings you define the size of the compose-"message box"

BB Code in Sigs
If switched on, BB-Code is allowed in the user signature

HTML in Sigs
If switched on, HTML-Code is allowed in the users signature

Impose a unique size to the picture signature
Here you can define a default size of picture signatures

Time Format
If the default time-setting in the forum is 24-hours or 12-hours. Each user can change this in his profile.

Date Format (mm/dd/yyyy, dd/mm/yy, etc.)
Define default date format for the forum. Each user can change this in his profile.

Global Unique Hour
If this is active, one time is used for the whole forum. Users are not allowed to use their own timezone.

Time Offset (GMT time is currently hh:mm)
Here you set the differenz between GMT-timezone and your local timezone in hours

Default-values for creating new Forums ... ( 8 options)
Here you can specify the default-values for creating new forums. These settings will only take effekt for NEW FORUMS, not for old once!

Default-values for creating new Members... ( 5 options)
Here you can specify the default-values for creating new members. These settings will only take effekt for NEW MEMBERS, not for old once! Please handle these settings with care (e.g. the email-adress of you members)!
